# Vinyl Store — Интернет-магазин виниловых пластинок

## 📁 Структура проекта

#### Защищённые маршруты:
| Маршрут | Метод | Доступ |
|---------|-------|--------|
| `/api/products` | GET | user, admin |
| `/api/products` | POST | admin |
| `/api/products/:id` | PUT | admin |
| `/api/products/:id` | DELETE | admin |
| `/api/admin/users` | GET | admin |
| `/api/admin/users/:id/role` | PATCH | admin |

#### Интерфейс:
- В шапке отображается роль пользователя
- Кнопки "Редактировать"/"Удалить" видны только администратору
- Кнопка "Добавить товар" видна только администратору

### Практика 7: Базовые методы аутентификации
- Регистрация пользователей с bcrypt хешированием паролей
- Вход в систему с проверкой пароля
- Валидация email и длины пароля

**Маршруты:**
- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход

---

### Практика 8: JWT токены
- При успешном входе выдаётся `accessToken` (15 минут) и `refreshToken` (7 дней)
- Защищённый маршрут `GET /api/auth/me` возвращает данные текущего пользователя
- Middleware `authMiddleware` проверяет валидность JWT

---

### Практика 9: Refresh-токены
- `POST /api/auth/refresh` — обновление пары токенов по refreshToken
- Автоматическая ротация refresh-токенов при обновлении

---

### Практика 10: Фронтенд + хранение токенов
- Страницы входа и регистрации
- Хранение токенов в `localStorage`
- Axios interceptors:
  - Автоматическая подстановка `Authorization: Bearer <token>`
  - При 401 — авто-обновление токенов через `/refresh` и повтор запроса
- Кнопка выхода с очисткой токенов
